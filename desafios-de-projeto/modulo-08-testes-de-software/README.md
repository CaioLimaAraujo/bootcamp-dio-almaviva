# Projeto V — Testes de Software (QA)

## Apresentação e Contextualização

Esse foi o último *Projeto de Código*, sendo parte do Módulo **Testes de Software**.

O projeto consistiu em uma simulação do dia a dia de um Analista QA em ambiente ágil (Scrum), com foco em testes manuais funcionais do tipo caixa-preta aplicados ao sistema [SwagLabs Shopping](https://www.saucedemo.com/) — uma loja virtual de demonstração amplamente utilizada em treinamentos de QA por conter cenários reais de teste.

O projeto cobre as funcionalidades de Login e Carrinho de Compras, passando por todas as etapas do processo de QA: levantamento de requisitos, escrita de User Stories, planejamento, documentação e elaboração de casos de teste. Todo o gerenciamento de projeto e a documentação foram desenvolvidos no JIRA e no Confluence, e exportados como PDFs para este repositório.

## Relatório Pessoal

Eu já tive experiências passadas com testes de software através da Formação Specialisterne, então para mim os dois últimos módulos foram bem legais para revisitar e reforçar alguns conceitos, quanto também para até mesmo aprofundar meus conhecimentos nessa área, e hoje em dia, com a era das IAs no desenvolvimento moderno de software, poder conciliar o uso da IA para auxiliar o desenvolvimento de software ao agilizar as partes mais monótonas do processo, com a aplicação de boas práticas e conhecimentos de QA/Teste de Software para garantir que o trabalho agilizado da IA está de acordo com os padrões de qualidade que são exigidos no mercado, foi uma experiência muito significante para mim. O desenvolvedor do futuro (e de hoje mesmo, na verdade) é aquele que sabe como usar a IA de maneira sensata, como uma ferramenta para agilizar e auxiliar o seu trabalho, e que sabe como averiguar se tudo está dentro dos conformes necessários de qualidade e segurança. 

Adaptação e agilidade, pensamento crítico e sensatez: habilidades cada vez mais necessárias na era moderna, seja na área da tecnologia ou em qualquer outro ramo profissional.
